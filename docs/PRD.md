# 마이링크 (MyLink) 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요
- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 사용자가 여러 소셜 미디어, 포트폴리오, 프로젝트, 웹사이트 등의 다양한 링크를 단일 페이지에 모아 간편하게 공유할 수 있도록 해주는 개인화된 멀티 링크 프로필 플랫폼.
- **대상 사용자**: 
  - 다양한 채널(인스타그램, 틱톡, 유튜브 등)을 함께 운영하는 크리에이터 및 인플루언서
  - 포트폴리오, 이력서, 깃허브(GitHub) 등을 한 번에 공유하고 싶은 개발자, 디자이너, 프리랜서, 취업 준비생
  - 별도의 구축 없이 간편하게 페이지를 홍보해야 하는 소상공인 및 소규모 브랜드

## 2. 핵심 기능 목록

### 🔴 필수 기능 (MVP - Minimum Viable Product)
| 기능명 | 설명 |
|---|---|
| **회원가입 및 로그인** | 이메일 기반 가입/로그인 및 소셜 로그인 지원. |
| **닉네임 기반 고유 URL** | 가입 시 설정한 닉네임을 기반으로 퍼블릭 프로필 주소 발급 (`mylink.com/nickname`). |
| **프로필 인라인 편집** | 별도의 화면 이동이나 모달 창 없이 즉시 닉네임과 짧은 자기소개(Bio)를 클릭하여 수정하는 **인라인 편집(Inline Edit)** 제공. (프로필 이미지는 구글 소셜 로그인의 프로필 사진을 가져와 사용하며, 직접 업로드 기능은 없음) |
| **링크 관리 및 파비콘(Favicon)** | 외부 URL을 입력하여 링크 추가. 추가 시 **구글(Google) 파비콘 API**를 호출하여 해당 도메인의 파비콘을 각 링크 아이템의 아이콘으로 자동 표시. |
| **링크 인라인 편집** | 등록된 링크의 제목과 URL을 항목 내에서 클릭하여 즉시 텍스트를 수정할 수 있는 인라인 편집 기능 지원. |
| **드래그 앤 드롭 정렬** | 추가된 링크 블록들을 사용자가 시각적으로 드래그 앤 드롭하여 우선순위를 직관적으로 변경 가능. |
| **실시간 프리뷰(Preview)**| 관리자 대시보드에서 링크나 프로필을 조작/수정할 때, 우측에 가상의 스마트폰 화면 형태로 어떻게 보이는지 즉시 확인 가능. |
| **기본 테마 지원** | 기본 라이트 모드/다크 모드 등 간단한 스타일 프리셋 제공. |

### 🟡 선택 기능 (Phase 2 & Advanced)
| 기능명 | 설명 |
|---|---|
| **트래픽 분석(Analytics)** | 총 방문자 조회수, 각 링크별 클릭 수, 클릭률(CTR) 등의 통계형 데이터 및 그래프 제공 |
| **SEO 및 오픈그래프 설정**| 사용자가 카카오톡 등에 프로필을 복사/공유 시 보여질 썸네일(OG Image) 타이틀을 설정 |
| **기간 한정 예약 링크** | 이벤트용으로 특정 시간에 맞춰 링크를 노출하고 숨기는 스케줄링 설정 기능 |

## 3. 각 기능 및 기획 상세 설명

### 3.1 사용자 및 프로필(Profile)
- **구글 프로필 이미지 연동**: 사용자가 직접 이미지를 업로드하는 대신, 구글 소셜 로그인 연동 시 구글 계정의 프로필 사진을 기본 프로필 이미지로 가져와 사용합니다.
- **닉네임 URL 규칙**: 닉네임은 URL(`/[닉네임]`)로 사용되므로, 띄어쓰기 제약 및 고유성(중복방지) 검증이 필요합니다. 닉네임 수정 시 기존 URL로의 접속이 불가해질 수 있습니다.

### 3.2 UI/UX : 인라인 편집 (Inline Editing)
- 사용자가 자신의 대시보드에서 수정하고 싶은 텍스트 영역(닉네임, 바이오, 링크 제목, 웹 주소 등)을 클릭하면 해당 부분이 즉각 입력 폼(Input)으로 변경되어 텍스트를 고칠 수 있습니다. 
- 입력을 마치고 포커스를 잃거나(Blur) 엔터를 누르면 실시간으로 저장이 수행(Auto-Save)되어 불필요한 '저장 버튼' 클릭을 최소화합니다.

### 3.3 아이콘 자동 적용 (구글 파비콘 API 활용)
- 사용자가 복잡하게 아이콘을 찾거나 업로드하지 않도록, 링크가 추가되면 다음과 같은 Google Favicon API를 사용하여 해당 링크의 공식 파비콘을 아이콘으로 자동 추출하여 표시합니다.
  - 예시: `https://s2.googleusercontent.com/s2/favicons?domain=[입력받은 URL]`

## 4. 데이터베이스 모델링 (ERD 설계 초안)

간단한 관계형 데이터베이스(RDBMS - MySQL, PostgreSQL 등)를 기준으로 설계한 초기 모델입니다. (Phase 1 기준)

### 1) Users (사용자 및 프로필 테이블)
사용자 계정과 프로필 정보를 통합하여 관리합니다.
| 컬럼명 | 타입 | 제약 조건 | 설명 |
|---|---|---|---|
| `id` | UUID / INT | PK | 고유 식별자 |
| `email` | VARCHAR | UNIQUE, NOT NULL | 가입 이메일 주소 |
| `password_hash` | VARCHAR | NOT NULL | 암호화된 비밀번호 |
| `nickname` | VARCHAR | UNIQUE, NOT NULL | 프로필 닉네임 (URL로 사용됨) |
| `profile_image_url` | VARCHAR | NULL | 구글 로그인 시 가져온 프로필 이미지 URL |
| `bio` | TEXT | NULL | 짧은 자기소개 |
| `created_at` | TIMESTAMP | DEFAULT NOW() | 계정 생성 일자 |
| `updated_at` | TIMESTAMP | DEFAULT NOW() | 정보 마지막 수정 일자 |

### 2) Links (개별 링크 아이템 테이블)
사용자가 등록한 여러 개의 링크 정보를 저장합니다. 회원(Users)과 1:N 관계를 가집니다.
| 컬럼명 | 타입 | 제약 조건 | 설명 |
|---|---|---|---|
| `id` | UUID / INT | PK | 링크 항목 고유 식별자 |
| `user_id` | UUID / INT | FK (Users.id) | 링크를 소유한 사용자 ID |
| `title` | VARCHAR | NOT NULL | 링크 버튼에 표시될 제목 |
| `url` | VARCHAR | NOT NULL | 이동할 목적지 링크 주소 |
| `favicon_url` | VARCHAR | NULL | (선택적) 구글 파비콘 API로 매핑된 아이콘 주소 |
| `order_index` | INT | NOT NULL, DEFAULT 0 | 드래그 앤 드롭 시 순서 정렬을 위한 값 |
| `is_active` | BOOLEAN | DEFAULT TRUE | 링크 표시 여부 (비활성화 시 화면에 숨김) |
| `created_at` | TIMESTAMP | DEFAULT NOW() | 링크 생성 시간 |
| `updated_at` | TIMESTAMP | DEFAULT NOW() | 링크 수정 시간 |
