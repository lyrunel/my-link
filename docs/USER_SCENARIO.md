# 마이링크 (MyLink) 사용자 시나리오 (User Scenarios)

> **목적**: 방문자와 페이지 소유자의 핵심 행동 흐름을 "사용자는 ___하기 위해 ___한다." 형식으로 정의합니다.

## 1. 방문자 (Visitor) 시나리오: 마이링크 페이지 방문 및 이용 과정

- 사용자는 **크리에이터의 여러 외부 채널(SNS, 포트폴리오 등)을 한 번에 확인하기 위해** 제공된 고유 URL(`mylink.com/nickname`)로 **접속한다**.
- 사용자는 **관심 있는 특정 콘텐츠나 사이트로 넘어가기 위해** 화면에 나열된 링크 버튼 중 하나를 **클릭하여 '새 탭(New Tab)'에서 원본 사이트로 이동한다**. (이탈 방지를 위해 모든 링크는 새 탭 열기를 기본으로 함)
- 사용자는 **해당 프로필을 지인에게 전달하기 위해** 페이지 내의 공유 버튼(또는 브라우저 주소창)을 통해 프로필 URL을 **복사하거나 공유한다**.
- 사용자는 **크리에이터가 설정해 둔 기간 한정 링크에 접근하기 위해** 이벤트 시간에 맞춰 동적으로 표시된 링크를 확인하고 **클릭한다**.

## 2. 소유자 (Owner) 시나리오: 링크 추가/수정/삭제 과정

### 2.1 링크 추가 파트
- 사용자는 **새로운 웹사이트를 방문자들에게 알리기 위해** 대시보드 화면(좌측)의 '링크 추가하기' 버튼을 **클릭한다**.
- 사용자는 **연결할 정확한 목적지를 등록하기 위해** 생성된 빈 폼의 URL 입력란에 외부 주소를 **입력(또는 붙여넣기) 한다**.
  *(시스템은 URL이 입력되면 해당 웹사이트의 기본 제목`<title>`과 구글 파비콘 아이콘을 자동으로 읽어와 채우며, 우측 프리뷰에 즉시 반영한다.)*

### 2.2 링크 및 프로필 수정 파트 (인라인 편집)
- 사용자는 **자동으로 불러온 링크 제목을 알아보기 쉽게 변경하기 위해** 링크 제목 텍스트를 직접 클릭하고 **내용을 수정한 뒤, 외부 공간을 클릭(Blur) 하거나 엔터를 눌러 즉시 적용한다**.
  *(시스템은 인라인 편집이나 정렬 등 UI 변경이 완료될 때마다 화면 하단에 가벼운 '스낵바(Snackbar) 알림'을 띄워 저장 완료를 사용자에게 인지시킨다.)*
- 사용자는 **연결 대상 주소가 바뀌었을 때 이를 갱신하기 위해** 링크의 기존 URL 텍스트를 클릭하여 **새로운 주소로 타이핑해 바꾼다**.
- 사용자는 **구글 로그인 시 가져온 기본 닉네임이나 소개말을 변경하기 위해** 상단의 프로필 영역 텍스트를 클릭하여 **자신만의 개성 있는 문구로 수정한다**.
- 사용자는 **가장 중요한 핵심 링크를 대시보드 최상단에 노출하기 위해** 해당 링크 블록의 좌측 그립(손잡이) 아이콘을 잡고 리스트 위로 **드래그 앤 드롭하여 정렬한다**.

### 2.3 링크 비활성화 및 삭제 파트
- 사용자는 **특정 이벤트가 끝난 링크를 영구 삭제하지 않고 당분간만 숨겨두기 위해** 해당 링크 항목 측면에 위치한 '상태 토글 스위치'를 클릭하여 **비활성화 상태로 변경한다**.
- 사용자는 **더 이상 쓸모가 없거나 잘못 등록한 링크를 완전히 지우기 위해** 휴지통(또는 삭제) 아이콘을 클릭하여 별도의 확인 창 없이 목록에서 **즉시 제거한다**. (빠른 관리 지원)
